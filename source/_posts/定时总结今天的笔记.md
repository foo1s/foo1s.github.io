# 🚀 OpenClaw + 本地 Obsidian 自动复盘部署指南 (2026.3 版本)
## 📌 核心思路
不需要复杂的 `mcporter` 中间件，也不需要折腾底层的向量数据库。利用 OpenClaw (配合 DeepSeek 等强力模型) 自带的 **文件操作 (File Operations)** 和 **系统执行 (Exec)** 工具，直接让 AI 去指定文件夹读取 `.md` 文件并进行总结。

---

## 🛠️ 第一阶段：准备工作
### 1. 确认大模型能力
确保你的 OpenClaw 配置了一个足够聪明的模型（推荐 DeepSeek-Chat、Claude 3.5 Sonnet 或 GPT-4o），因为系统原生的文件检索强依赖模型的理解能力。

- **验证方法：** 在 OpenClaw 聊天框输入：`你有读取本地文件的能力吗？`
- 只要它回答“有”，就可以继续。

### 2. 获取 Obsidian 的绝对路径
找到你的 Obsidian 笔记库在电脑上的完整路径。

- **注意：** 必须把所有的反斜杠 `\` 替换为正斜杠 `/`！
- **错误示例：** `C:\Users\24589\Documents\Obsidian`
- **正确示例：** `C:/Users/24589/Documents/Obsidian` _(这是为了防止 OpenClaw 在发送 JSON 数据包时发生转义错误崩溃。)_

---

## ⚙️ 第二阶段：设置定时任务 (Cron Job)
这是最核心的一步，全部在网页端图形化界面完成。

1. **打开控制台：** 在终端运行 `openclaw dashboard`，打开网页管理后台。
2. **进入自动化菜单：** 点击左侧边栏的 **Automation** (自动化) -> 选择 **Cron Jobs**。
3. **创建新任务：** 在右侧的 **New Job** 表单中填写信息：
    
    - **Name (名称):** `Daily Obsidian Recap` (或任意名称)
    - **Agent ID:** 输入并选中 `main`，确保旁边的复选框是 `Enabled`。
        
4. **设置时间 (Schedule):**
    
    - 点击最上方的下拉菜单（默认可能是 `Every`）。
    - 选择 **`Cron`** (自定义表达式)。
    - 在出现的输入框中填入：**`0 23 * * *`** (代表每天 23:00)。
    - **Timezone (时区):** 在下拉框中手动输入并选择 **`Asia/Shanghai`**。
        
5. **设置执行模式 (Execution):**
    
    - **Session:** 选择 `Isolated` (隔离模式，这样它会在后台默默执行，不会打断你当前的聊天)。
    - **Wake mode:** 选择 `Now`。
        
6. **输入核心指令 (What should run? / Prompt):** 把下面这段话复制进去（**务必替换为你第一阶段准备好的带正斜杠 `/` 的真实路径**）：
    
    > “现在是每晚复盘时间。我的 Obsidian 笔记库的具体路径是：`C:/Users/24589/Documents/Obsidian`。 请使用你的【文件操作】或【exec】工具，找出这个目录下今天（按文件修改时间）新建或修改过的所有 `.md` 文件。 逐个读取这些文件的内容，提取出核心知识点、主要进展和待办事项，最后用清晰的排版（如 Markdown 列表）给我生成一份今日的学习复盘总结。”
    
7. **保存并启用：** 滚动到最下方，点击 **Create** 或 **Save**。

---

## 🎯 第三阶段：测试与验收
不需要傻等几个小时看它报不报错，直接手动测试一次：

1. 在 **Cron Jobs** 列表中，找到你刚才建好的 `Daily Obsidian Recap`。
2. 点击任务卡片右下角的 **`Run`** 按钮。
3. 去左侧菜单切换到 **Chat (聊天)** 界面。
4. 如果你看到 AI 已经开始输出带有你今天笔记内容（比如 Dubbo、Kafka）的精美复盘报告……

**🎉 恭喜！你的本地 AI 知识库自动化链路已永久打通！** 以后每天晚上 23:00，它都会准时为你送上当天的学习战报。

# 🚀 自动化本地 AI 知识库 (OpenClaw + Obsidian) 技术复盘报告
## 一、 项目执行摘要 (Executive Summary)
本项目成功构建了一条跨操作系统（Windows Host <-> WSL Ubuntu）的全自动 AI 笔记处理链路。利用 OpenClaw (v2026.3.13) 作为核心调度网关，结合大模型（DeepSeek）的原生系统操作能力，实现了每天 23:00 (Asia/Shanghai) 自动读取本地 Obsidian 增量笔记，并生成结构化技术复盘报告的完整闭环。

## 二、 核心技术栈 (Tech Stack)
- **大脑与调度层：** OpenClaw (2026.3.13) Gateway & UI Dashboard
- **大语言模型：** Custom DeepSeek API (`deepseek-chat`)
- **底层运行环境：** Windows (Host) + WSL Ubuntu (QMD 服务端)
- **数据源与格式：** 本地 Obsidian Vault (`.md` Markdown 文件)

---

## 三、 关键战役与技术攻坚点 (Key Milestones & Troubleshooting)
在整个部署过程中，我们一共排除了 4 个极其隐蔽的底层技术障碍：

### 1. 协议与端点对齐 (Protocol & Endpoint Alignment)
- **挑战：** 试图通过 MCP (Model Context Protocol) 桥接本地 WSL 服务时，通信握手失败（状态一直显示 offline）。
- **破局：** 摒弃了常规的 `/sse` 流式后缀，通过查阅底层运行日志，精准定位到 QMD 在当前版本使用的是 `/mcp` 专属端点。修正通信地址为 `http://127.0.0.1:8181/mcp` 后，成功实现底层物理连通。

### 2. 配置文件严格校验 (JSON Strictness & Schema Validation)
- **挑战：** 尝试通过直接修改 `openclaw.json` 来注入工具时，遭遇了致命的 `SyntaxError: JSON5: invalid character '}'` 以及后续的 `Unrecognized keys: "mcpServers"`。
- **破局：** 1. 使用了 OpenClaw 原生的 `openclaw doctor --fix` 命令完成了配置文件的“外科手术式”回滚与修复。
    
    2. **重要认知转变：** 明确了 OpenClaw 采用的是沙盒化的 Agent 工具链管理（依赖原生系统工具或外部生态包），而非 Claude 桌面版的原生 `mcpServers` 字段逻辑。

### 3. “大道至简”的架构降维 (Architecture Pivot)
- **挑战：** 依赖 `mcporter` 中间件和 QMD 向量检索来做“每日总结”，不仅链路长，而且容易因为跨系统通信导致断流。
- **破局：** 发现当前挂载的 DeepSeek 模型配合 OpenClaw 的 `[文件操作]` 和 `[exec]` 原生工具，已经具备极强的底层读写能力。我们果断放弃了复杂的中间件路由，直接采用**“绝对路径精准定向”**的策略，让 AI 直捣黄龙读取 Markdown 原文件。效率和稳定性直接拉满。

### 4. 数据包转义与 JSON 崩溃 (The Path Escaping Fatal Error)
- **挑战：** 在最后一步执行 Automation Cron Job 时，触发了 `JSON Parse error`。
- **破局：** 这是一个经典的跨平台数据结构陷阱。Windows 环境下的文件路径反斜杠 `\` 在封装成 JSON 发送给底层执行器时，被误认为了“转义符”（Escape Character），导致 JSON 结构破裂。
- **修复方案：** 将提示词中的 Windows 路径（`C:\Users\...`）全部替换为正斜杠格式（`C:/Users/...`），彻底消灭了语法冲突，使任务成功运转。

---

## 四、 最终运行机制 (The Final Pipeline)
现在，你拥有了一个极致优雅的系统。每天晚上 23:00，系统会默默执行以下流程：

1. **触发器：** OpenClaw Cron 引擎根据 `0 23 * * * (Asia/Shanghai)` 准时唤醒 `main` Agent。
2. **隔离运行：** 开启一个 `Isolated` 会话，不干扰你前端的任何工作。
3. **系统调用：** Agent 携带安全路径 `C:/Users/24589/Documents/Obsidian`，通过 `exec` 探针检索今日存在修改的 `.md` 文件。
4. **读取与总结：** AI 读取你的代码笔记（Dubbo、Kafka、Redis等），提取知识密度最高的核心部件，生成你截图中的结构化战报。
